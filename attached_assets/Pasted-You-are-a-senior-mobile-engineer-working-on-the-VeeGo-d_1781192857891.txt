Fix 1 — Driver Must Broadcast Location During Shuttle Trip (Critical)
Problem: The driver app currently sends location updates for on-demand rides only. For shuttle trips, the backend expects the driver to broadcast their location so passengers can see them on the map 20 minutes before departure.
What to do:
When the driver goes online and has a shuttle trip departing within 20 minutes:
Start emitting DRIVER_LOCATION_UPDATE socket event continuously (every 3 seconds).
Payload: { tripId, lat, lng, heading }
This is the same event already used for on-demand rides — just make sure it also fires for shuttle trips.
Stop emitting when the trip status becomes completed or cancelled.
Do not emit location for shuttle trips that are more than 20 minutes away.
Fix 2 — Selfie Check-In at Trip Start (Critical)
Problem: The driver app has a selfie check-in screen but it only triggers after 10 hours of continuous online time. For shuttle, the backend sends shuttle:checkin:required when the driver goes online within 3 hours of a shuttle trip. The app does not listen to this event.
What to do:
Add a listener for socket event: shuttle:checkin:required
Payload: { tripId, deadlineMinutes: 10 }
When received:
Immediately navigate the driver to the existing selfie check-in screen.
Show a countdown timer of 10 minutes on the screen.
If the driver submits the selfie within 10 minutes via the existing POST /driver/checkin endpoint, add tripId to the request body.
If the timer runs out without submission, show a warning: "لم تقم بالتحقق في الوقت المحدد. لن تظهر للركاب حتى تقوم بالتحقق."
Block the driver from starting the trip until the selfie is submitted.
Fix 3 — Station Boarding Timer (60 Seconds)
Problem: The backend sends shuttle:station:timeout after 60 seconds if the driver has not handled all passengers at a station. The app does not listen to this event.

Add a listener for socket event: shuttle:station:timeout
Payload: { tripId, stationId }

This is a reminder only — do not auto-advance the driver.
Fix 4 — Send stationId When Boarding Passengers
Problem: The boarding endpoint POST /shuttle/bookings/:id/board accepts an optional stationId that triggers the 60-second station timer on the backend. The app currently does not send this field.

Include stationId in the body of every POST /shuttle/bookings/:id/board call.
The stationId should come from the current station the driver is at in the trip route.
