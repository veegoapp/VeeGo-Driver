# Full Backend Integration & Feature Ownership Audit

Perform a COMPLETE deep audit of the entire Passenger App codebase.

Your goal is to identify:

1. Everything that communicates with the backend.
2. Everything that depends on backend APIs.
3. Everything that receives backend data.
4. Everything that sends data to the backend.
5. Everything that exists only in the Passenger App.
6. Everything that is actually controlled by the Driver App.
7. Everything that is controlled by Admin Dashboard.
8. Any feature that appears in the Passenger App UI but is not connected to a backend endpoint.
9. Any backend endpoint that exists but is never used by the Passenger App.
10. Any screens, buttons, actions, notifications, sockets, realtime events, hooks, queries, mutations, contexts, services, or stores that are disconnected from the backend.

Generate a professional markdown report named:

PASSENGER_APP_BACKEND_AUDIT.md

The report must be exhaustive and must not skip any screen, feature, API, hook, context, service, state manager, notification flow, booking flow, shuttle flow, wallet flow, chat flow, profile flow, authentication flow, or realtime flow.
