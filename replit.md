# VeeGo Driver

React Native (Expo) app for VeeGo ride-hailing drivers ‚Äî manages online status, ride offers, active trip tracking, earnings, and live surge zone visibility.

## Run & Operate

- **Workflow (`VeeGo Driver`)** ‚Äî runs `setup.sh` automatically on start:
  1. Checks `BACKEND_URL` secret is set (exits with error if missing)
  2. Pings `$BACKEND_URL/health` and warns if backend is unreachable
  3. Writes `.env` ‚Üí `EXPO_PUBLIC_API_URL=$BACKEND_URL`
  4. Runs `pnpm install`
  5. Starts `pnpm exec expo start --tunnel --clear`
- `pnpm run typecheck` ‚Äî full typecheck

## Stack

- Expo SDK 53 / React Native, Expo Router, pnpm
- Maps: MapLibre GL JS via `react-native-webview` (native) and `maplibre-gl` (web)
- API client: `lib/api.ts` ‚Äî reads `EXPO_PUBLIC_API_URL` at bundle time
- Sockets: `socket.io-client` in `hooks/useRideSocket.ts` ‚Äî strips `/api` suffix for socket URL

## Where things live

- `lib/api.ts` ‚Äî all REST endpoints; base URL from `EXPO_PUBLIC_API_URL`
- `hooks/useRideSocket.ts` ‚Äî WebSocket connection; derives socket URL from `EXPO_PUBLIC_API_URL`
- `components/MapBackdrop.native.tsx` ‚Äî native map (WebView + MapLibre HTML, OSRM routing, bearing indicator)
- `components/MapBackdrop.web.tsx` ‚Äî web map (MapLibre GL JS directly)
- `setup.sh` ‚Äî startup script run by the Replit workflow
- `.env` ‚Äî auto-generated by `setup.sh`; never edit manually

## Architecture decisions

- Native map uses `react-native-webview` rendering inline MapLibre HTML instead of `react-native-maps` (incompatible with Expo Go SDK 53)
- `expo-notifications` is dynamically required with try/catch guards ‚Äî it is removed from Expo Go SDK 53; the app degrades gracefully (no push tokens in Expo Go)
- `EXPO_PUBLIC_API_URL` is the single source of truth for both REST base URL and socket URL derivation
- `pnpm exec expo start` (not `npx expo start`) ensures the locally installed Expo version is used

## User preferences

- `setup.sh` must: (1) check `BACKEND_URL` secret, (2) ping backend health, (3) write `.env`, (4) `pnpm install`, (5) `pnpm exec expo start --tunnel --clear`
- Use `pnpm exec expo start` not `npx expo start`

## Gotchas

- `BACKEND_URL` secret must be set in Replit Secrets before the workflow will start. The app shows "Cannot reach the server" if the backend repl is asleep or the URL is wrong.
- `EXPO_PUBLIC_*` vars are baked in at Metro bundle time ‚Äî changing `.env` requires `--clear` to take effect (already included in the start command).
- The socket URL is derived by stripping `/api` from `EXPO_PUBLIC_API_URL` in `useRideSocket.ts`.
- Do NOT use `npx expo start` ‚Äî it may pull a different CLI version than the installed `@expo/cli`.
