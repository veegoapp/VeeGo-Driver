Fix Driver App post-login routing logic.

----------------------------------------------------
CURRENT ISSUE
----------------------------------------------------
After every login, users are ALWAYS redirected to:
→ /service-select

This happens even for returning drivers who already completed onboarding.

Root cause:
login.tsx contains a hardcoded redirect:
router.replace('/service-select')

No check exists for previously selected service state.

----------------------------------------------------
REQUIRED FIX
----------------------------------------------------

1 — AFTER LOGIN SUCCESS
Do NOT immediately redirect to /service-select.

Instead:

- Check AsyncStorage for:
  veego_service_type

----------------------------------------------------
2 — ROUTING LOGIC
----------------------------------------------------

IF veego_service_type EXISTS:
    → route directly to /(tabs) or main driver dashboard

IF veego_service_type DOES NOT EXIST:
    → route to /service-select (first-time onboarding only)

----------------------------------------------------
3 — SINGLE RESPONSIBILITY RULE
----------------------------------------------------
login.tsx must ONLY:
- store tokens
- complete authentication

It MUST NOT decide navigation flow.

Create a separate post-auth routing decision logic layer if needed.

----------------------------------------------------
4 — EXPECTED FLOW
----------------------------------------------------

First signup:
login → no service_type → service-select → save selection → dashboard

Returning user:
login → service_type exists → dashboard مباشرة

----------------------------------------------------
5 — DATA SOURCE
----------------------------------------------------
Use ONLY:
- AsyncStorage key: veego_service_type

Do NOT add backend dependency.

----------------------------------------------------
6 — DO NOT CHANGE
----------------------------------------------------
- Do not modify backend
- Do not introduce new auth fields
- Do not remove serviceContext persistence

----------------------------------------------------
EXPECTED RESULT
----------------------------------------------------
- Service selection appears ONLY once per account
- Returning drivers go directly to dashboard
